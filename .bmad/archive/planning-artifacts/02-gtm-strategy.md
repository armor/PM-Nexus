# Nexus UI Uplift - GTM Strategy

> **Version:** 1.0
> **Last Updated:** 2026-01-03
> **Status:** Draft
> **Primary Goal:** AMP usage drops to zero by end of Q1

---

## 1. Executive Summary

This Go-To-Market (GTM) strategy outlines the approach to successfully transition all customers from AMP to Nexus by end of Q1, ensuring seamless adoption, high satisfaction, and continued trust in Armor's platform.

### Key Objectives
1. **Zero AMP Usage** by Q1 end
2. **Customer Retention** - No churn attributed to migration
3. **Sales Enablement** - Demo-ready environments
4. **Support Readiness** - Self-service capability

---

## 2. Market Context

### 2.1 Current State
- AMP is the legacy platform customers rely on
- Nexus exists as POC with limited production readiness
- Customers have varying levels of AMP dependency
- Sales teams rely on AMP for demonstrations

### 2.2 Target State
- Nexus is the sole customer platform
- All AMP features available (or improved) in Nexus
- Sales demos exclusively on Nexus
- Customer self-service without support escalations

## 3. Customer Segmentation

### 3.1 Migration Cohorts

| Cohort | Description | Size | Priority | Migration Window |
|--------|-------------|------|----------|------------------|
| **Early Adopters** | Tech-forward, minimal AMP dependency | ~15% | P1 | Weeks 1-4 |
| **Core Users** | Regular AMP users, standard workflows | ~60% | P2 | Weeks 5-10 |
| **Power Users** | Heavy AMP dependency, custom workflows | ~20% | P3 | Weeks 11-13 |
| **Enterprise** | Complex integrations, SLAs | ~5% | P4 | White-glove |

### 3.2 Cohort-Specific Strategies

**Early Adopters:**
- Beta access to new features
- Direct feedback channel
- Case study opportunities
- Recognition in release notes

**Core Users:**
- Guided migration with tutorials
- Office hours for questions
- Email campaign with tips
- In-app migration assistant

**Power Users:**
- Dedicated CSM assignment
- Custom migration planning
- Feature parity validation
- Extended parallel access

**Enterprise:**
- Executive sponsor alignment
- Custom integration support
- SLA review and commitment
- Dedicated migration team

## 7. Customer Success Playbook

### 7.1 Migration Steps

```
1. Pre-Migration
   ├── Account health check
   ├── Feature usage analysis
   ├── Stakeholder identification
   └── Migration timeline agreement

2. Migration Execution
   ├── Nexus account provisioning
   ├── User access setup
   ├── Data verification
   ├── Integration testing
   └── Parallel running period

3. Post-Migration
   ├── AMP access removal
   ├── Training completion check
   ├── Feedback collection
   └── Success metrics review
```

### 7.2 Customer Touchpoints

| Touchpoint | Timing | Owner | Goal |
|------------|--------|-------|------|
| Migration kickoff | Week -1 | CSM | Align on timeline |
| Account setup | Day 1 | CSM | Enable access |
| Training session | Day 3 | CSM | Onboard users |
| Check-in call | Day 7 | CSM | Address issues |
| Parallel review | Day 14 | CSM | Validate parity |
| Cutover confirmation | Day 21 | CSM | Confirm ready |
| Success review | Day 30 | CSM | Measure adoption |

### 7.3 Escalation Path

```
Level 1: CSM (< 4 hours)
    │
    ├─── Resolved → Document & Close
    │
    ▼
Level 2: CS Manager (< 24 hours)
    │
    ├─── Resolved → Root cause analysis
    │
    ▼
Level 3: Product/Engineering (< 48 hours)
    │
    ├─── Resolved → Feature/Fix shipped
    │
    ▼
Level 4: Executive (Critical only)
```
